# 项目开发规范

## 1. 概述

本文档定义了企业级并发与安全示例项目的开发规范，旨在保证代码质量、提高开发效率、降低维护成本。

## 2. 项目结构规范

### 2.1 目录结构

```
src/main/java/com/example/concurrent_impl/
├── common/                    # 公共模块
│   ├── config/               # 配置类
│   ├── constant/             # 常量定义
│   ├── enums/                # 枚举定义
│   ├── exception/            # 异常处理
│   ├── response/             # 响应封装
│   └── util/                 # 工具类
├── entity/                    # 实体类
│   └── enums/                # 实体枚举
├── mapper/                    # 数据访问层
├── service/                   # 业务逻辑层
├── controller/                # 接口层
├── highconcurrency/          # 高并发场景
│   ├── threadsafety/         # 线程安全
│   ├── lock/                 # 锁机制
│   │   └── service/          # 锁服务
│   ├── ratelimit/            # 限流
│   │   ├── annotation/       # 限流注解
│   │   ├── aspect/           # 限流切面
│   │   └── service/          # 限流服务
│   └── cache/                # 缓存策略
│       └── service/          # 缓存服务
├── businesssecurity/         # 业务安全
│   ├── idempotent/           # 幂等
│   │   ├── annotation/       # 幂等注解
│   │   └── aspect/           # 幂等切面
│   ├── validation/           # 参数校验
│   │   └── validator/        # 自定义校验器
│   └── permission/           # 权限控制
├── transactionsafety/        # 事务安全
│   ├── local/                # 本地事务
│   ├── distributed/          # 分布式事务
│   │   └── service/          # 分布式事务服务

    ├── strong/               # 强一致性
    └── versioning/           # 版本控制
```

### 2.2 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 包名 | 全小写，单词间用点分隔 | `com.example.concurrent_impl` |
| 类名 | 大驼峰 | `OrderService` |
| 方法名 | 小驼峰 | `createOrder()` |
| 常量 | 全大写，下划线分隔 | `MAX_RETRY_COUNT` |
| 变量 | 小驼峰 | `userId` |
| 参数 | 小驼峰 | `productId` |

## 3. 代码规范

### 3.1 类注释规范

```java
/**
 * 【模块名称】类的功能描述
 *
 * 【业务背景】
 * 描述该类的业务场景和使用背景
 *
 * 【实现原理】
 * 描述该类的核心实现原理
 *
 * 【为什么这样写】
 * 描述设计决策的原因
 *
 * 【不遵守的后果】
 * 描述不遵循规范可能导致的问题
 *
 * 【正确示例】
 * 提供正确的使用示例
 *
 * 【错误示例】
 * 提供错误的使用示例
 *
 * 【实际案例】
 * 提供实际业务场景
 *
 * @author 作者名
 * @date 创建日期
 */
```

### 3.2 方法注释规范

```java
/**
 * 方法功能描述
 *
 * 【实现原理】
 * 描述方法的实现原理
 *
 * @param paramName 参数说明
 * @return 返回值说明
 * @throws ExceptionType 异常说明
 */
```

### 3.3 代码格式规范

- 缩进：使用4个空格
- 行宽：不超过120字符
- 空行：方法之间空一行
- 导入：按包分组，组间空一行

## 4. Git规范

### 4.1 分支命名

| 类型 | 格式 | 示例 |
|------|------|------|
| 功能分支 | `feature/功能描述` | `feature/order-create` |
| 修复分支 | `bugfix/问题描述` | `bugfix/lock-timeout` |
| 热修复 | `hotfix/问题描述` | `hotfix/security-fix` |
| 发布分支 | `release/版本号` | `release/1.0.0` |

### 4.2 提交信息规范

```
<type>(<scope>): <subject>

<body>

<footer>
```

**类型(type)**：
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具相关

**示例**：
```
feat(order): 添加订单创建接口

- 实现订单创建逻辑
- 添加分布式锁防重复下单
- 添加限流保护

Closes #123
```

## 5. 数据库规范

### 5.1 表命名

- 全小写，单词间用下划线分隔
- 表名使用复数形式
- 示例：`users`, `orders`, `products`

### 5.2 字段命名

- 全小写，单词间用下划线分隔
- 主键：`id`
- 外键：`表名_id`（如 `user_id`）
- 创建时间：`created_at`
- 更新时间：`updated_at`
- 删除标记：`deleted`（逻辑删除）

### 5.3 索引命名

- 主键索引：`pk_表名`
- 唯一索引：`uk_表名_字段名`
- 普通索引：`idx_表名_字段名`

## 6. API规范

### 6.1 URL命名

- 使用RESTful风格
- 全小写，单词间用连字符分隔
- 使用名词复数形式
- 示例：`/api/orders`, `/api/users/{id}`

### 6.2 HTTP方法

| 方法 | 用途 | 示例 |
|------|------|------|
| GET | 查询 | `GET /api/orders` |
| POST | 创建 | `POST /api/orders` |
| PUT | 全量更新 | `PUT /api/orders/{id}` |
| PATCH | 部分更新 | `PATCH /api/orders/{id}` |
| DELETE | 删除 | `DELETE /api/orders/{id}` |

### 6.3 响应格式

```json
{
    "code": 200,
    "message": "success",
    "data": {}
}
```

## 7. 异常处理规范

### 7.1 异常分类

| 异常类型 | 说明 | 处理方式 |
|----------|------|----------|
| BusinessException | 业务异常 | 返回错误信息给用户 |
| SystemException | 系统异常 | 记录日志，返回通用错误信息 |
| ThirdPartyException | 第三方异常 | 记录日志，重试或降级 |

### 7.2 错误码规范

- 成功：200
- 客户端错误：4xx
- 服务端错误：5xx
- 业务错误：6xx

## 8. 日志规范

### 8.1 日志级别

| 级别 | 用途 |
|------|------|
| ERROR | 系统错误，需要立即处理 |
| WARN | 警告信息，需要关注 |
| INFO | 业务流程信息 |
| DEBUG | 调试信息，生产环境关闭 |

### 8.2 日志内容

- 时间戳
- 线程名
- 日志级别
- 类名
- 日志内容
- 关键参数

## 9. 测试规范

### 9.1 测试分类

| 类型 | 说明 | 覆盖率要求 |
|------|------|------------|
| 单元测试 | 测试单个方法 | >= 80% |
| 集成测试 | 测试模块间交互 | >= 60% |
| 端到端测试 | 测试完整流程 | 关键流程100% |

### 9.2 测试命名

```java
@Test
public void test_方法名_场景_期望结果() {
    // Given
    // When
    // Then
}
```

## 10. 安全规范

### 10.1 敏感信息处理

- 密码：使用BCrypt加密
- 手机号：脱敏显示
- 身份证：脱敏显示
- 银行卡：脱敏显示

### 10.2 接口安全

- 使用HTTPS
- 参数校验
- SQL注入防护
- XSS防护
- CSRF防护
- 接口限流
