# 架构设计文档

## 1. 项目概述

### 1.1 项目简介

本项目是一个企业级并发与安全代码实现示例，展示了在高并发、分布式环境下的各种解决方案和最佳实践。

### 1.2 技术栈

| 类别 | 技术 | 版本 |
|------|------|------|
| 基础框架 | Spring Boot | 3.5.x |
| 编程语言 | Java | 17 |
| 数据库 | MySQL | 8.x |
| 缓存 | Redis | 7.x |
| 消息队列 | RabbitMQ | 3.x |
| ORM | Spring Data JPA | - |
| 文档 | SpringDoc OpenAPI | 2.x |
| 构建工具 | Maven | 3.x |

## 3. 模块设计

### 3.1 高并发模块

#### 3.1.1 分布式锁

**类图**：
```
RedisLockService
├── tryLock(lockKey, timeout): String
├── tryLockWithRetry(lockKey, timeout, maxRetries, interval): String
├── unlock(lockKey, lockValue): boolean
├── isLocked(lockKey): boolean
└── executeWithLock(lockKey, timeout, task): T
```

**时序图**：
```
Client -> RedisLockService: tryLock()
RedisLockService -> Redis: SET NX EX
Redis --> RedisLockService: OK/FAIL
RedisLockService --> Client: lockValue/null

Client -> RedisLockService: executeWithLock()
RedisLockService -> Redis: SET NX EX
Redis --> RedisLockService: OK
RedisLockService -> Task: execute()
Task --> RedisLockService: result
RedisLockService -> Redis: DEL (Lua)
RedisLockService --> Client: result
```

#### 3.1.2 限流服务

**支持的限流算法**：
1. 滑动窗口限流
2. 固定窗口限流
3. 令牌桶限流

**类图**：
```
RedisRateLimitService
├── slidingWindowRateLimit(key, windowMs, limit): boolean
├── fixedWindowRateLimit(key, limit, windowSeconds): boolean
├── tokenBucketRateLimit(key, capacity, rate): boolean
└── getCurrentCount(key): long

@RateLimit (注解)
├── prefix: String
├── windowMs: long
├── limit: int
└── type: LimitType

RateLimitAspect
└── before(joinPoint, rateLimit): void
```

#### 3.1.3 缓存保护

**支持的防护策略**：
1. 防穿透：缓存空值
2. 防击穿：互斥锁
3. 防雪崩：随机过期时间

**类图**：
```
CacheProtectionService
├── getWithPenetrationProtection(key, dbQuery, expire): String
├── getWithBreakdownProtection(key, dbQuery, expire): String
└── getWithAvalancheProtection(key, dbQuery, baseExpire, randomExpire): String
```

### 3.2 业务安全模块

#### 3.2.1 幂等控制

**类图**：
```
@Idempotent (注解)
├── key: String (SpEL)
├── expireSeconds: int
├── message: String
├── prefix: String
└── deleteKeyAfterFailure: boolean
```

**流程图**：
```
开始 -> 生成幂等键 -> 检查Redis
                         │
         ┌───────────────┼───────────────┐
         ▼               ▼               ▼
      不存在           存在且未过期      存在且已过期
         │               │               │
         ▼               ▼               ▼
    设置幂等键        返回错误         删除旧键
         │                              │
         ▼                              ▼
    执行业务逻辑                     设置新幂等键
         │                              │
         ▼                              ▼
    检查执行结果                     执行业务逻辑
         │
    ┌────┴────┐
    ▼         ▼
  成功       失败
    │         │
    ▼         ▼
  保留键    删除键
    │         │
    └────┬────┘
         ▼
       结束
```

### 3.3 事务安全模块

#### 3.3.1 本地消息表

**表结构**：
```sql
CREATE TABLE local_message (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    message_id VARCHAR(64) NOT NULL UNIQUE,
    exchange VARCHAR(100) NOT NULL,
    routing_key VARCHAR(100) NOT NULL,
    content TEXT,
    status INT NOT NULL DEFAULT 0,
    retry_count INT DEFAULT 0,
    max_retry_count INT DEFAULT 3,
    next_retry_time DATETIME,
    error_message VARCHAR(500),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

**流程图**：
```
业务操作 -> 保存消息到本地表 -> 提交事务
                                    │
                                    ▼
                            定时任务扫描消息
                                    │
                                    ▼
                            发送消息到MQ
                                    │
                        ┌───────────┼───────────┐
                        ▼           ▼           ▼
                     发送成功     发送失败     超过重试次数
                        │           │           │
                        ▼           ▼           ▼
                    更新状态     增加重试次数   标记失败
                   为已发送     设置下次重试
```

## 5. 缓存设计

### 5.1 缓存策略

| 数据类型 | 缓存策略 | 过期时间 | 更新策略 |
|----------|----------|----------|----------|
| 用户信息 | Cache-Aside | 30分钟 | 写时删除 |
| 商品信息 | Cache-Aside | 10分钟 | 写时删除 |
| 订单信息 | 不缓存 | - | - |
| 配置信息 | Read-Through | 1小时 | 定时刷新 |

### 5.2 缓存Key设计

```
用户信息：user:{userId}
商品信息：product:{productId}
分布式锁：lock:{lockKey}
幂等键：idempotent:{prefix}:{key}
限流键：rate_limit:{type}:{key}
```

## 7. 安全设计

### 7.1 认证授权

- 使用JWT Token认证
- 基于RBAC的权限控制
- 接口级别权限校验

### 7.2 数据安全

- 密码使用BCrypt加密
- 敏感数据脱敏存储
- 传输层使用HTTPS
- 数据库连接使用SSL

### 7.3 接口安全

- 参数校验
- SQL注入防护
- XSS防护
- CSRF防护
- 接口限流
- 幂等控制
