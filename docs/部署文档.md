# 部署文档

## 1. 环境要求

### 1.1 服务器配置

| 环境 | CPU | 内存 | 磁盘 | 数量 |
|------|-----|------|------|------|
| 开发环境 | 2核 | 4GB | 50GB | 1 |
| 测试环境 | 4核 | 8GB | 100GB | 1 |
| 生产环境 | 8核 | 16GB | 200GB | 2+ |

### 1.2 软件依赖

| 软件 | 版本 | 说明 |
|------|------|------|
| JDK | 17 | OpenJDK或Oracle JDK |
| MySQL | 8.x | 主从架构 |
| Redis | 7.x | 哨兵或集群模式 |
| RabbitMQ | 3.x | 集群模式 |
| Nginx | 1.x | 负载均衡 |

## 2. 环境配置

### 2.1 开发环境配置

**application-dev.yml**：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/concurrent_demo?useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: root
  
  data:
    redis:
      host: localhost
      port: 6379
  
  rabbitmq:
    host: localhost
    port: 5672
    username: guest
    password: guest
```

### 2.2 生产环境配置

**application-prod.yml**：
```yaml
spring:
  datasource:
    url: jdbc:mysql://master:3306/concurrent_demo?useSSL=true&serverTimezone=Asia/Shanghai
    username: ${DB_USERNAME}
    password: ${DB_PASSWORD}
    hikari:
      maximum-pool-size: 50
      minimum-idle: 10
  
  data:
    redis:
      sentinel:
        master: mymaster
        nodes: sentinel1:26379,sentinel2:26379,sentinel3:26379
      password: ${REDIS_PASSWORD}
  
  rabbitmq:
    host: rabbitmq-cluster
    port: 5672
    username: ${MQ_USERNAME}
    password: ${MQ_PASSWORD}
```

## 3. 数据库部署

### 3.1 创建数据库

```sql
CREATE DATABASE concurrent_demo DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 3.2 创建用户

```sql
CREATE USER 'app_user'@'%' IDENTIFIED BY 'strong_password';
GRANT ALL PRIVILEGES ON concurrent_demo.* TO 'app_user'@'%';
FLUSH PRIVILEGES;
```

### 3.3 执行初始化脚本

```bash
mysql -u root -p concurrent_demo < src/main/resources/db/schema.sql
```

## 4. Redis部署

### 4.1 单机部署

```bash
# 安装Redis
sudo apt-get install redis-server

# 配置Redis
sudo vim /etc/redis/redis.conf
# 设置密码
requirepass your_password

# 启动Redis
sudo systemctl start redis
sudo systemctl enable redis
```

### 4.2 哨兵部署

**sentinel.conf**：
```conf
sentinel monitor mymaster 127.0.0.1 6379 2
sentinel down-after-milliseconds mymaster 5000
sentinel failover-timeout mymaster 60000
sentinel auth-pass mymaster your_password
```

## 5. RabbitMQ部署

### 5.1 安装RabbitMQ

```bash
# 安装Erlang
sudo apt-get install erlang

# 安装RabbitMQ
sudo apt-get install rabbitmq-server

# 启用管理插件
sudo rabbitmq-plugins enable rabbitmq_management

# 启动RabbitMQ
sudo systemctl start rabbitmq-server
sudo systemctl enable rabbitmq-server
```

### 5.2 创建用户和虚拟主机

```bash
# 创建用户
sudo rabbitmqctl add_user app_user strong_password

# 设置用户标签
sudo rabbitmqctl set_user_tags app_user administrator

# 创建虚拟主机
sudo rabbitmqctl add_vhost /concurrent_demo

# 设置权限
sudo rabbitmqctl set_permissions -p /concurrent_demo app_user ".*" ".*" ".*"
```

## 6. 应用部署

### 6.1 打包应用

```bash
# 清理并打包
mvn clean package -DskipTests

# 打包结果
ls -la target/concurrent_impl-*.jar
```

### 6.2 启动应用

```bash
# 开发环境
java -jar target/concurrent_impl-*.jar --spring.profiles.active=dev

# 生产环境
java -jar target/concurrent_impl-*.jar \
  --spring.profiles.active=prod \
  --server.port=8080 \
  -Xms4g -Xmx4g \
  -XX:+UseG1GC \
  -XX:MaxGCPauseMillis=100
```

### 6.3 Systemd服务配置

**/etc/systemd/system/concurrent-impl.service**：
```ini
[Unit]
Description=Concurrent Impl Application
After=syslog.target network.target

[Service]
User=app
Group=app
ExecStart=/usr/bin/java -jar /opt/concurrent_impl/concurrent_impl.jar \
  --spring.profiles.active=prod
SuccessExitStatus=143
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
# 启用服务
sudo systemctl enable concurrent-impl
sudo systemctl start concurrent-impl
sudo systemctl status concurrent-impl
```

## 7. Nginx配置

### 7.1 负载均衡配置

```nginx
upstream concurrent_impl {
    server 192.168.1.10:8080 weight=1;
    server 192.168.1.11:8080 weight=1;
    keepalive 32;
}

server {
    listen 80;
    server_name api.example.com;

    location / {
        proxy_pass http://concurrent_impl;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_connect_timeout 30s;
        proxy_read_timeout 60s;
        proxy_send_timeout 60s;
    }

    # 限流配置
    limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s;

    location /api/orders {
        limit_req zone=api burst=20 nodelay;
        proxy_pass http://concurrent_impl;
    }
}
```

## 8. Docker部署

### 8.1 Dockerfile

```dockerfile
FROM openjdk:17-jdk-slim

WORKDIR /app

COPY target/concurrent_impl-*.jar app.jar

EXPOSE 8080

ENTRYPOINT ["java", "-jar", "app.jar"]
```

### 8.2 docker-compose.yml

```yaml
version: '3.8'

services:
  app:
    build: .
    ports:
      - "8080:8080"
    environment:
      - SPRING_PROFILES_ACTIVE=prod
      - DB_HOST=mysql
      - DB_USERNAME=root
      - DB_PASSWORD=root
      - REDIS_HOST=redis
      - RABBITMQ_HOST=rabbitmq
    depends_on:
      - mysql
      - redis
      - rabbitmq

  mysql:
    image: mysql:8.0
    ports:
      - "3306:3306"
    environment:
      - MYSQL_ROOT_PASSWORD=root
      - MYSQL_DATABASE=concurrent_demo
    volumes:
      - mysql_data:/var/lib/mysql

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

  rabbitmq:
    image: rabbitmq:3-management
    ports:
      - "5672:5672"
      - "15672:15672"

volumes:
  mysql_data:
```
